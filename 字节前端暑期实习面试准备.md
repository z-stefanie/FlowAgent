## 二、JavaScript 核心（占比约 30%，必考）

### 2.1 事件循环 — 出现率最高，几乎每场必考

**经典输出题（务必能秒答）：**

```javascript
async function async1() {
  console.log('async1 start');
  await async2();
  console.log('async1 end');
}
async function async2() {
  console.log('async2');
}

console.log('script start');

setTimeout(function() {
  console.log('setTimeout');
}, 0);

async1();

new Promise(function(resolve) {
  console.log('promise1');
  resolve();
}).then(function() {
  console.log('promise2');
});

console.log('script end');
```

**输出顺序：**
```
script start → async1 start → async2 → promise1 → script end → async1 end → promise2 → setTimeout
```

**核心知识点：**
- **宏任务（MacroTask）：** setTimeout、setInterval、I/O、UI rendering、MessageChannel
- **微任务（MicroTask）：** Promise.then/catch/finally、MutationObserver、queueMicrotask、process.nextTick（Node）
- **执行顺序：** 同步代码 → 微任务队列清空 → 宏任务（取一个） → 微任务队列清空 → 循环
- `await` 后面的代码等价于 `.then()` 回调，属于微任务
- `new Promise(executor)` 中 executor 是同步执行的
- **浏览器 vs Node.js 事件循环区别：** Node.js 有 6 个阶段（timers / pending / idle / poll / check / close），`process.nextTick` 优先级高于 `Promise.then`

### 2.2 闭包

**核心定义：** 函数能够访问其词法作用域（定义时所在的作用域），即使该函数在其词法作用域之外执行。

**常见考点：**
- 闭包导致的内存泄漏问题：引用未释放的变量。解决方法：手动置 null、使用 WeakRef
- for 循环 + setTimeout 经典输出题（let vs var 的区别）
- 实际应用：防抖节流、柯里化、模块化模式、React Hooks 闭包陷阱

### 2.3 this 指向

四种绑定规则（优先级从低到高）：
1. **默认绑定**：非严格模式 → window，严格模式 → undefined
2. **隐式绑定**：`obj.fn()` 中 this → obj
3. **显式绑定**：call / apply / bind
4. **new 绑定**：this → 新创建的实例

**箭头函数：** 继承外层 this，无法被 call/apply/bind 改变，没有 arguments、prototype，不能 new

### 2.4 Promise

- **三种状态：** pending → fulfilled / rejected（不可逆）
- async/await 本质是 Generator + 自动执行器的语法糖
- **请求竞态问题：** 使用 AbortController 取消过期请求，或用自增 ID 标记最新请求

### 2.5 数据类型与类型判断

**基本类型（7种）：** number、string、boolean、undefined、null、symbol、bigint

**引用类型：** object（含 Array、Function、Date、RegExp、Map、Set 等）

**判断方法：**
```javascript
typeof val                          // 基本类型（null 除外），函数返回 'function'
val instanceof Ctor                 // 沿原型链查找
Object.prototype.toString.call(val) // 最准确，'[object Type]'
Array.isArray(val)                  // 专门判断数组
```

**经典面试题：** `==` vs `===`
- `==` 会进行类型转换（对象调用 `valueOf()` → `toString()` → 转原始值）
- 经典题：让 `a == 1 && a == 2 && a == 3` 为 true → 重写 `valueOf` 或用 `Proxy`

### 2.6 原型与原型链

- 每个对象都有 `__proto__` 指向其构造函数的 `prototype`
- 原型链的终点是 `Object.prototype.__proto__ === null`
- `instanceof` 的原理：沿原型链查找 `right.prototype`
- ES5 继承（寄生组合式）vs ES6 `class extends`

### 2.7 V8 垃圾回收机制（高频新增）

| 区域 | 算法 | 特点 |
|------|------|------|
| 新生代 | Scavenge（复制算法） | 空间换时间，From/To 两个半区 |
| 老生代 | Mark-Sweep + Mark-Compact | 标记清除 + 标记整理，减少碎片 |

- **WeakMap/WeakSet vs Map/Set：** WeakMap 的 key 是弱引用，不影响 GC 回收，适合存储 DOM 节点引用等场景，解决内存泄漏问题

### 2.8 ES6+ 重要特性

- let/const vs var（块级作用域、暂时性死区、不会变量提升到全局）
- 解构赋值、展开运算符、模板字符串
- Map/Set/WeakMap/WeakSet
- Proxy/Reflect（Vue3 响应式基础）
- 可选链 `?.`、空值合并 `??`
- Iterator / Generator

## 三、CSS & HTML（占比约 15%）

### 3.1 水平垂直居中（几乎每场必考，至少说出 4 种）

```css
/* 方案一：Flex（最推荐） */
.parent { display: flex; justify-content: center; align-items: center; }

/* 方案二：Grid */
.parent { display: grid; place-items: center; }

/* 方案三：绝对定位 + transform */
.child { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); }

/* 方案四：绝对定位 + margin auto（需已知宽高） */
.child { position: absolute; top: 0; right: 0; bottom: 0; left: 0; margin: auto; width: 200px; height: 200px; }
```

### 3.2 BFC（块格式化上下文）

**触发条件：**
- `overflow` 不为 visible / clip
- `display: flex / inline-block / grid / table / flow-root`
- `position: absolute / fixed`
- float 不为 none

**应用：** 清除浮动、防止 margin 重叠、自适应两栏布局

### 3.3 盒模型

- `content-box`（标准）：width = content
- `border-box`（IE）：width = content + padding + border
- 推荐全局 `*, *::before, *::after { box-sizing: border-box; }`

### 3.4 CSS 选择器优先级

`!important` > 内联(1000) > ID(100) > 类/伪类/属性(10) > 元素/伪元素(1) > 通配符(0)

### 3.5 重排与重绘

- **重排（Reflow）：** 改变几何属性（width/height/margin/padding/display/position）
- **重绘（Repaint）：** 改变外观属性（color/background/visibility）
- **transform/opacity 做动画为什么流畅？** 不触发重排重绘，由 GPU 在合成层（Compositor Layer）直接处理，跳过 Layout 和 Paint 阶段

### 3.6 Flex 布局

- `flex: 1` 是 `flex-grow: 1; flex-shrink: 1; flex-basis: 0%` 的简写
- `flex-grow` 分配剩余空间，`flex-shrink` 计算收缩比例
- 三栏布局：左右固定宽，中间 `flex: 1`

### 3.7 文本溢出处理

```css
/* 单行 */
.single { white-space: nowrap; overflow: hidden; text-overflow: ellipsis; }

/* 多行（WebKit） */
.multi {
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

### 3.8 响应式设计

- `rem`：相对于根元素 font-size
- `em`：相对于父元素 font-size
- `vw/vh`：相对于视口宽/高的 1%
- 媒体查询：`@media (max-width: 768px) { ... }`

## 四、浏览器与网络（占比约 20%，必考）

### 4.1 从输入 URL 到页面展示（经典必考，深度追问）

1. **DNS 解析**：浏览器缓存 → 系统缓存 → 路由器缓存 → ISP DNS → 递归/迭代查询
2. **TCP 三次握手**：SYN → SYN+ACK → ACK
3. **TLS 握手**（HTTPS）：证书验证、密钥交换（非对称加密协商对称密钥）
4. **发送 HTTP 请求**：请求行、请求头、请求体
5. **服务器处理并返回响应**
6. **浏览器解析与渲染**：
   - HTML → DOM 树
   - CSS → CSSOM 树
   - DOM + CSSOM → Render Tree
   - Layout（布局）→ 分层（Layer）→ 分块（Tiling）→ **光栅化（Rasterization，由 GPU 执行）** → 合成（Composite）→ 显示
7. **TCP 四次挥手断开连接**

> **深度追问：** 为什么光栅化要由 GPU 做？— GPU 擅长并行计算像素颜色值，大幅加速位图生成。合成阶段将各层位图合成最终画面并显示。

### 4.2 浏览器渲染 — 什么会阻塞 DOM 构建？

- `<script>` 标签（无 defer/async）**阻塞** DOM 解析
- CSS **不阻塞** DOM 解析，但**阻塞渲染**（CSSOM 构建完才能合成 Render Tree）
- `<script defer>` 不阻塞 DOM，在 DOMContentLoaded 前按序执行
- `<script async>` 不阻塞 DOM，下载完立即执行（不保证顺序）

### 4.3 HTTP 缓存机制

| 类型 | 字段 | 优先级 | 说明 |
|------|------|--------|------|
| 强缓存 | Cache-Control: max-age | 最高 | 直接用本地缓存，不发请求 |
| 强缓存 | Expires | 次高 | 绝对时间，受本地时间影响 |
| 协商缓存 | ETag / If-None-Match | 中 | 文件指纹，精确到内容变化 |
| 协商缓存 | Last-Modified / If-Modified-Since | 低 | 精确到秒，可能不准确 |

**禁止缓存：** `Cache-Control: no-store`（完全不缓存）vs `no-cache`（每次都协商）

### 4.4 跨域问题

- **原因：** 浏览器同源策略（协议+域名+端口必须完全一致）
- **CORS：** 服务端设置 `Access-Control-Allow-Origin`
  - **简单请求：** GET/HEAD/POST + 特定 Content-Type，直接发送
  - **复杂请求：** 先发 OPTIONS 预检请求（Preflight），服务端返回允许的方法和头
- **代理服务器：** 开发环境常用（Vite proxy / Nginx 反向代理）
- **JSONP：** 仅 GET，利用 `<script>` 不受同源限制，已过时
- **postMessage：** 跨窗口/iframe 通信

### 4.5 HTTP/2 与 HTTP/3

- **HTTP/2 多路复用**：一个 TCP 连接并发多个请求/响应，解决 HTTP 层队头阻塞
- **HTTP/3（QUIC）**：基于 UDP，解决 TCP 层面的队头阻塞，0-RTT 连接

### 4.6 HTTPS 原理

- 非对称加密（RSA/ECDHE）交换对称密钥 → 对称加密（AES）传输数据
- CA 证书验证服务端身份，防止中间人攻击
- **为什么不全用非对称加密？** 非对称加密计算开销大，对称加密传输更高效

### 4.7 Cookie / Storage

| 特性 | Cookie | localStorage | sessionStorage |
|------|--------|-------------|---------------|
| 容量 | ~4KB | ~5MB | ~5MB |
| 生命周期 | 可设置过期时间 | 永久 | 页面关闭清除 |
| 请求携带 | 自动携带 | 不携带 | 不携带 |
| 作用域 | 可跨子域 | 同源 | 同源+同标签页 |

**Cookie 安全属性：**
- `HttpOnly`：禁止 JS 访问，防 XSS 窃取
- `Secure`：仅 HTTPS 传输
- `SameSite`：防 CSRF（Strict / Lax / None）

### 4.8 TCP 与 UDP

| TCP | UDP |
|-----|-----|
| 面向连接 | 无连接 |
| 可靠传输（确认/重传/排序） | 不可靠，可能丢包 |
| 有流量控制和拥塞控制 | 无 |
| 头部 20 字节 | 头部 8 字节 |
| HTTP/1.1、HTTP/2 | DNS、HTTP/3(QUIC)、视频直播 |

**四次挥手为什么要等待 2MSL？** 确保最后的 ACK 能到达对方；确保本连接的所有报文在网络中消失，不影响新连接。

### 4.9 前端性能优化与指标

**核心 Web Vitals：**
- **LCP（Largest Contentful Paint）**：最大内容绘制，< 2.5s
- **FID / INP（Interaction to Next Paint）**：交互延迟，< 200ms
- **CLS（Cumulative Layout Shift）**：累计布局偏移，< 0.1

**首屏优化手段：**
- 代码分割（路由懒加载）、图片懒加载、DNS 预解析（`<link rel="dns-prefetch">`）
- `preload`（预加载关键资源）/ `prefetch`（预获取将来资源）
- SSR / SSG、CDN 加速、Gzip/Brotli 压缩

**运行时优化：**
- 防抖节流、虚拟列表、减少重排重绘
- Web Worker 处理 CPU 密集型任务
- `requestAnimationFrame` 替代 setTimeout 做动画

**排查工具：** Lighthouse、Chrome DevTools Performance 面板、Web Vitals 库

## 五、React 深度拷问（占比约 20%，字节最看重！）

### 5.1 Fiber 架构

- **本质：** 将递归的虚拟 DOM diff 改为**链表结构**的可中断工作单元
- **遍历方式：** 深度优先、先序遍历（child → sibling → return）
- **目的：** 让渲染可以暂停、恢复，优先处理高优先级更新（如用户输入）
- **协调阶段（Reconciliation）可中断，提交阶段（Commit）不可中断**
- **调度实现：** 用 `MessageChannel` 而非 `requestIdleCallback`（因为 rIC 触发不稳定，且在后台标签页不执行）

### 5.2 Diff 算法

- **三个核心假设：** ① 不同类型元素产生不同树 ② 开发者可用 key 暗示哪些子元素稳定 ③ 只做同层比较
- **key 的作用：** 帮助 React 识别元素身份，复用 DOM 节点
- **为什么不能用数组索引做 key？** 插入/删除/排序时索引变化，导致错误复用和不必要的重渲染

### 5.3 Hooks 原理

- **本质：** Hooks 以链表形式存储在 Fiber 节点的 `memoizedState` 上
- **为什么不能在 if/for 中调用？** 靠调用顺序匹配状态，条件/循环会打乱链表顺序
- **useEffect vs useLayoutEffect：** useEffect 在浏览器绘制后异步执行；useLayoutEffect 在 DOM 更新后、浏览器绘制前同步执行（适合读取 DOM 布局信息）
- **useEffect 闭包陷阱：** effect 内捕获的是创建时的 state 快照。解决：使用 ref 保存最新值，或在 deps 中正确声明依赖
- **常用 Hooks：** useState、useEffect、useRef、useMemo、useCallback、useContext、useReducer、useId

### 5.4 React 版本演进

| React 16 | React 18 | React 19（新热点） |
|-----------|----------|-------------------|
| Fiber 架构引入 | 并发特性（Concurrent） | **React Compiler**：自动记忆化 |
| 合成事件系统 | 自动批处理 | **`use` API**：原生支持 Promise 数据获取 |
| - | `startTransition` | **Server Actions**：组件中直接调用后端函数 |
| - | `useSyncExternalStore` | **Server Components (RSC)**：组件在服务端运行，0 JS 发送到客户端 |
| - | Suspense SSR 流式渲染 | 消灭 useMemo/useCallback/React.memo 的手动优化 |

> **RSC vs SSR 区别：** SSR 是在服务端把组件渲染成 HTML 字符串；RSC 是组件逻辑完全在服务端执行，客户端不下载该组件的 JS 代码，只接收序列化的渲染结果。

### 5.5 虚拟 DOM

- 本质是 JS 对象树（不是真实 DOM）
- 作用：跨平台抽象、batch 更新、diff 计算最小变更
- React diff 策略：同层比较 → key 匹配 → 类型不同直接替换

### 5.6 状态管理 — Zustand（字节常考）

- 在 React 外维护 store（发布-订阅模式）
- 通过 `useSyncExternalStore` 把外部状态安全同步到组件
- 相比 Redux：无 boilerplate、无 Provider、体积更小
- 适用场景：中小型应用全局状态、跨组件共享

### 5.7 React 性能优化

- `React.memo` / `useMemo` / `useCallback`（React 19 后可能不再需要手动写）
- **虚拟列表**（react-window / react-virtuoso）— 不定高虚拟列表：预估高度 + 动态测量 + 缓存
- 代码分割（React.lazy + Suspense）
- `startTransition` 降低更新优先级
- 避免不必要的重渲染（key 正确设置、状态下沉、组件拆分）

### 5.8 受控组件 vs 非受控组件

- **受控组件：** 表单值由 React state 驱动，每次输入都触发 onChange → setState
- **非受控组件：** 用 ref 直接读取 DOM 值，适合文件上传等场景
- 推荐受控组件（数据流更可控），复杂表单可用 react-hook-form（非受控 + ref 方案，性能更好）

## 六、Vue 相关（如果项目用 Vue）

### 6.1 Vue2 vs Vue3 响应式

| Vue2 | Vue3 |
|------|------|
| Object.defineProperty | Proxy |
| 无法检测属性新增/删除（需 $set） | 完全支持 |
| 数组需特殊处理（重写 7 个方法） | 原生支持 |
| 嵌套对象递归劫持（初始化开销大） | 惰性代理（访问时才递归，性能更好） |

### 6.2 Vue3 Composition API vs Options API

- Composition API 逻辑复用更好（替代 mixins），代码按功能组织而非按选项组织
- 更好的 TypeScript 支持
- `<script setup>` 编译时优化

## 七、TypeScript（上升趋势）

### 7.1 字节真题：实现 DeepReadonly

```typescript
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? T[K] extends Function
      ? T[K]
      : DeepReadonly<T[K]>
    : T[K];
};
```

### 7.2 常考类型工具（手写实现）

```typescript
type MyPartial<T> = { [K in keyof T]?: T[K] };

type MyRequired<T> = { [K in keyof T]-?: T[K] };

type MyPick<T, K extends keyof T> = { [P in K]: T[P] };

type MyOmit<T, K extends keyof T> = Pick<T, Exclude<keyof T, K>>;
```

### 10.1 防抖（Debounce）— 支持立即执行 + 取消

```javascript
function debounce(fn, delay, immediate = false) {
  let timer = null;
  const debounced = function(...args) {
    if (timer) clearTimeout(timer);
    if (immediate && !timer) {
      fn.apply(this, args);
    }
    timer = setTimeout(() => {
      if (!immediate) fn.apply(this, args);
      timer = null;
    }, delay);
  };
  debounced.cancel = () => { clearTimeout(timer); timer = null; };
  return debounced;
}
```

### 10.3 深拷贝（处理循环引用 + 特殊类型）

```javascript
function deepClone(obj, map = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (obj instanceof Date) return new Date(obj);
  if (obj instanceof RegExp) return new RegExp(obj);
  if (map.has(obj)) return map.get(obj);

  const clone = Array.isArray(obj) ? [] : {};
  map.set(obj, clone);

  for (const key of Reflect.ownKeys(obj)) {
    clone[key] = deepClone(obj[key], map);
  }
  return clone;
}
```

### 10.7 带并发限制的异步调度器（2026 超高频！）

```javascript
class Scheduler {
  constructor(limit) {
    this.limit = limit;
    this.running = 0;
    this.queue = [];
  }

  add(task) {
    return new Promise(resolve => {
      const wrappedTask = () => task().then(resolve);
      if (this.running < this.limit) {
        this._run(wrappedTask);
      } else {
        this.queue.push(wrappedTask);
      }
    });
  }

  _run(task) {
    this.running++;
    task().then(() => {
      this.running--;
      if (this.queue.length > 0) {
        this._run(this.queue.shift());
      }
    });
  }
}
```

```javascript
  list.forEach(item => map.set(item.id, { ...item, children: [] }));

    const node = map.get(item.id);
```

```javascript
function parseQuery(queryStr) {
```

### 10.11 LRU 缓存（LC 146，二面高频算法）

```javascript
class LRUCache {
  constructor(capacity) {
    this.capacity = capacity;
    this.cache = new Map();
  }

  get(key) {
    if (!this.cache.has(key)) return -1;
    const val = this.cache.get(key);
    this.cache.delete(key);
    this.cache.set(key, val);
    return val;
  }

  put(key, value) {
    if (this.cache.has(key)) this.cache.delete(key);
    this.cache.set(key, value);
    if (this.cache.size > this.capacity) {
      this.cache.delete(this.cache.keys().next().value);
    }
  }
}
```

### 10.13 发布订阅模式（EventEmitter）

```javascript
class EventEmitter {
  constructor() { this.events = new Map(); }

  on(event, fn) {
    const fns = this.events.get(event) || [];
    fns.push(fn);
    this.events.set(event, fns);
  }

  off(event, fn) {
    const fns = this.events.get(event);
    if (fns) this.events.set(event, fns.filter(f => f !== fn));
  }

  emit(event, ...args) {
    (this.events.get(event) || []).forEach(fn => fn(...args));
  }

  once(event, fn) {
    const wrapper = (...args) => { fn(...args); this.off(event, wrapper); };
    this.on(event, wrapper);
  }
}
```

```javascript
function fetchWithRetry(url, options = {}, retries = 3, delay = 1000) {
  return new Promise((resolve, reject) => {
    const attempt = (n) => {
      fetch(url, options)
        .then(resolve)
        .catch(err => {

  });
}
```
